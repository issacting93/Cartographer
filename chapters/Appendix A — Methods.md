## A.1 Corpus Construction

### v1 Corpus (Phase 1)

**Source datasets:** WildChat (Zhao et al. 2024), Chatbot Arena (LMSYS), OASST (Open Assistant).

**Sampling:** 625 conversations drawn from public datasets. Selection criteria: English language, minimum 4 messages, at least one complete human-AI exchange pair.

**Deduplication:** SHA-256 content hashing removed 98 exact duplicates (content-identical conversations appearing under different IDs) and 20 near-duplicates, yielding a canonical corpus of **N=507**.

**Lesson learned:** Deduplication was not performed in the initial analysis. The 98 duplicates inflated apparent pattern strength and were detected only during the v1 audit (Chapter 5, Instance 1).

### v2 Corpus (Phase 2)

**Source datasets:** Same three sources, expanded sampling.

| Source | Count | % |
|--------|-------|---|
| Chatbot Arena | 1,739 | 67.5% |
| WildChat | 786 | 30.5% |
| OASST | 50 | 1.9% |
| Other | 2 | 0.1% |
| **Total** | **2,577** | |

**Deduplication:** SHA-256 applied at ingestion. Manifest stored in `data/v2_unified/manifests/dedupe_manifest.json`.

**Descriptive statistics:** Median conversation length: 6 messages. Mean: 10.1.

### Mental Health Corpus (Phase 4)

**Single-turn (N=100):** Amod/mental_health_counseling_conversations dataset (HuggingFace). Real licensed therapist responses to patient questions. One question, one answer. Downloaded via `scripts/download_mental_health.py`.

**Multi-turn (N=50 pilot):** MentalChat16K PISCES subset (HuggingFace). Behavioral health coach conversations with caregivers. Mean 7.6 turns per session. **Caveat:** This data is synthetic, generated by GPT-3.5. Cross-model comparability with Claude-classified analysis is not validated.

**Full PISCES dataset:** 423 sessions available; 50 processed in pilot. Scaling to full dataset is planned.

## A.2 Role Classification (SRT)

### Taxonomy

12 roles grounded in Social Role Theory (Parsons 1951, Bales 1950, Eagly 1987):

**Human roles (6):** Information-Seeker, Provider, Director, Collaborator, Social-Expressor, Relational-Peer

**AI roles (6):** Expert-System, Advisor, Co-Constructor, Learning-Facilitator, Social-Facilitator, Relational-Peer

Each role is positioned along two axes:
- **Instrumental ↔ Expressive** (task-oriented vs. relationship-oriented)
- **Authority level** (high, equal, low)

### Classification Method

LLM-based classification via `scripts/classify_roles_srt.py`. Each conversation is classified into:
- Dominant human role (from distribution)
- Dominant AI role (from distribution)
- 8 interaction dimensions (emotional tone, interaction pattern, conversation purpose, topic depth, etc.)

**Models used:**
- v1: GPT-5.2 (OpenAI)
- v2: GPT-5.2 (OpenAI) for main corpus; Claude Haiku (`claude-haiku-4-5-20251001`) for mental health extension
- Model adapter: `scripts/atlas/anthropic_adapter.py` wraps Anthropic SDK to OpenAI interface

### Instrumental/Expressive Classification

A role is classified as **instrumental** if it is Information-Seeker, Provider, or Director (human side) or Expert-System, Advisor, or Learning-Facilitator (AI side). All others are **expressive**.

- v1 (N=507): 98.8% instrumental human roles (verified from `data/v1_canonical/reports/taxonomy_comprehensive.json`)
- v2 (N=2,576 with roles): 97.0% instrumental (2,498/2,576; verified from `verified_stats.json`)

## A.3 Evidence Feature Architecture

### The Circularity Problem

In v1, role-derived spatial features were used as evidence for role importance, producing R²=1.0 — a tautology (Chapter 5, Instance 1). This forced a strict separation:

### Feature Separation (Enforced)

| Feature Type | Count | Source | Use | R² with labels |
|-------------|-------|--------|-----|---------------|
| Evidence features | 30 | Text-derived (affect, divergence, dynamics, expressiveness, structure) | All empirical claims | < 0.01 |
| Viz-only features | 2 | Role-derived spatial coordinates | Terrain rendering only | N/A (by construction) |

### Evidence Feature Channels

| Channel | Features | Contribution to RF |
|---------|----------|-------------------|
| Affect | affect_mean, affect_trend, affect_min, affect_variance, affect_range, affect_max, affect_peak_count, affect_valley_count, valence_mean, valence_trend, valence_variance | 39.3% |
| Divergence | div_mean, div_trend, div_max_spike, div_range, div_variance | 21.6% |
| Dynamics | goal_stability, goal_drift_mean, goal_drift_variance, constraint_pressure, hedge_assert_ratio, ai_refusal_rate, repair_rate | 16.5% |
| Expressiveness | expr_mean, expr_trend, expr_variance, expr_shift, expr_range | 15.4% |
| Structure | length_ratio, n_messages_log | 7.2% |

Channel contributions verified from `verified_stats.json → rf_role_pair.channel_contributions`.

### Acceptance Tests

Three formal acceptance tests gate all quantitative claims:

- **Test A:** Evidence features must have R² < 0.01 with role labels
- **Test B:** No role-derived features appear in any empirical claim
- **Test C:** Statistical claims match computed values within rounding tolerance

Implementation: `scripts/run_acceptance_tests.py`. Status: A/B/C PASS.

## A.4 Interaction Graph Pipeline

A 7-stage computational pipeline transforms raw conversations into annotated interaction graphs:

| Stage | Component | Method |
|-------|-----------|--------|
| 1 | Move Classifier | Hybrid: regex patterns (unambiguous cases) + LLM fallback (ambiguous). 13 communicative act types. |
| 2 | Mode Detector | Classifies turn interaction mode: LISTENER, ADVISOR, EXECUTOR. *Note: v1 taxonomy; v2 6-stance taxonomy proposed but not yet implemented (see Chapter 4).* |
| 3 | Constraint Tracker | Deterministic state machine. Identifies user constraints, tracks AI compliance, monitors repair. |
| 4 | Graph Builder | NetworkX MultiDiGraph. Nodes: turns, moves, constraints. Edges: violations, repairs, mode transitions. |
| 5 | Metrics Computer | 8 CUI diagnostic metrics per conversation. |
| 6 | Visualization | Polar trajectory plots, 3D terrain maps. |
| 7 | Export | Standardized JSON. |

**Implementation:** `scripts/atlas/run_pipeline.py`

### Constraint Tracking

The constraint tracker is the empirical backbone of the Agency Collapse findings.

**Detection:** User constraints identified by deontic markers, explicit requirements, and stated preferences.

**State machine:** Each constraint transitions through states: ACTIVE → SURVIVED or VIOLATED. Violated constraints may trigger REPAIR_ATTEMPTED → REPAIR_SUCCEEDED or REPAIR_FAILED.

**Results (atlas_canonical, N=745 graphs, 376 constraints):**

| Metric | Value | Verification |
|--------|-------|-------------|
| Total constraints | 376 | verified_stats.json |
| Violation rate | 71.5% (269/376) | verified_stats.json |
| Survival rate | 28.5% (107/376) | verified_stats.json |
| Mean time-to-violation | 2.5 turns | verified_stats.json |
| Median time-to-violation | 1 turn | verified_stats.json |
| Turn 0 violations | 24.5% (66/269) | verified_stats.json |
| Repair events | 271 | verified_stats.json |

## A.5 Agency Collapse Definition

Agency Collapse is defined as a **binary outcome variable**, independent of role classification or clustering:

A conversation exhibits collapse if **any** of these conditions hold:

1. **Repair Failure:** repair_count ≥ 5 with persistent misalignment
2. **Tone Degradation:** stance_delta > 0.5 AND politeness_delta < -0.2
3. **Specificity Collapse:** specificity_delta < -1.0 AND passive_rate > 0.4

**Rate:** 50.3% (200/398 sustained conversations with sufficient turns for trajectory analysis).

**Threshold sensitivity:** Conversations below 3 repair attempts mostly recover. Conversations crossing 5 attempts collapse at 89.1% rate. The intervention window is narrow.

## A.6 Random Forest Validation

To verify the taxonomy captures real structure (not noise), a Random Forest classifier was trained on the 16 most common role-pair classes using **only** the 30 evidence features:

| Parameter | Value |
|-----------|-------|
| Target | Role pair (16 classes) |
| Features | 30 evidence features (no role-derived) |
| N | 2,525 conversations |
| CV folds | 5 |
| Estimators | 200 |
| **Accuracy** | **71.6%** (chance = 6.2%) |

This confirms that role pairs produce genuinely different conversational signatures. The 28.4% error rate is reported honestly: the taxonomy compresses reality; it doesn't define it.

## A.7 PAD Affect Measurement

**v1:** Pleasure-Arousal-Dominance values had only 53 unique values across 507 conversations — too coarse for reliable affect analysis.

**v2:** Improved affect proxy produces 503 unique values. Computed per message via `data/v2_unified/affect_proxy/`. Values stored in conversation JSON as `messages[].pad` with keys `p`, `a`, `d`.

**Variance ratio computation:** For IS→ES pairs with ≥6 messages, compute per-conversation affect variance. Ratio of maximum to minimum variance: 2,817x (smoothest: chatbot_arena_11761, variance 7.37e-06; most volatile: wildchat_new_300289e3c40bce07, variance 0.0208).

## A.9 Limitations

1. **No human-human baseline.** All findings describe human-AI interaction. Whether the instrumental monopoly or constraint decay rates differ in human-human text chat is unknown (Phase 5 planned).

2. **LLM-as-classifier.** Role classification, move classification, and mode detection all rely on LLM judgments. Inter-rater reliability between human coders and LLM classifiers has not been formally established.

3. **Corpus bias.** WildChat and Chatbot Arena over-represent technically oriented users and English-language conversations.

4. **Synthetic mental health data.** The multi-turn PISCES subset (N=50) was generated by GPT-3.5. Cross-model comparability is unvalidated.

5. **Mode taxonomy v1.** The LISTENER/ADVISOR/EXECUTOR taxonomy is demonstrably inadequate for non-task-oriented domains (Chapter 4). The v2 6-stance taxonomy is proposed but not implemented.

6. **Formative study provenance.** The N=20 formative evaluation of the Context Inventory Interface is reported as completed, but raw data has not been archived and specific numbers vary between documents. These results should be treated as preliminary.
